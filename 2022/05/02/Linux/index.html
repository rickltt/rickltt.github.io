  • cd ../dir 切换到上一级目录的dir目录下。

一般我们登入Linux系统时会启动一个bash shell，会在几个启动文件中查找命令，有两种启动方式：

  1. 登录式shell ：正常通过某终端登录的 shell。

bash的配置文件：

  • 全局：/etc/profile 、/etc/profile.d/*.sh 、/etc/bashrc
  • 局部： ~/.bash_profile 、~/.bashrc

登录式shell读取配置文件过程：

/etc/profile –> /etc/profile.d/*.sh –> ~/.bash_profile–> ~/.bashrc –> /etc/bashrc

非登录式shell读取配置文件过程：

~/.bashrc –> /etc/bashrc –> /etc/prodile.d/*.sh

如果想要全局环境变量持久化，最好是在/etc/profile.d目录中创建一个以.sh结尾的文件。把所有新的或修改过的全局环境变量设置放在这个文件中。

如果是个人用户的局部环境变量持久化，需要修改的是~/.bashrc文件。

添加环境变量就是在文件末尾加上，export <环境变量名>=<路径>

文件权限

每个Linux文件具有四种访问权限：可读(r)、可写(w)、可执行(x)和无权限(-)。
利用ls -l命令可以看到某个文件或目录的权限，它以显示数据的第一个字段为准。第一个字段由10个字符组成，如下：

-rwxr-xr-x

  • 第一位表示文件类型，-表示文件，d表示目录
  • 后面有三组rwx，分别代表 文件所有者（Owner）、用户组（Group）、其它用户（Other Users）的权限。

修改文件权限

数字表示法： 是指将r、w和x分别用4、2、1来代表，没有授予权限的则为0，然后把权限相加

文本表示法：4个字母表示不同的用户

  • u：所有者
  • g：组成员
  • o：其他成员
  • a：所有人

权限仍用r、w和x表示

  • =：重新制定权限
  • -：对目前的设置减少权限
  • +：对目前的设置增加权限

Linux组

用户账户在控制单个用户安全性方面很好用，但涉及在共享资源的一组用户时就捉襟见肘了。为了解决这个问题，Linux系统采用了另外一个安全概念——组（group）。

组权限允许多个用户对系统中的对象（比如文件、目录或设备等）共享一组共用的权限。

/etc/group

与用户账户类似，组信息也保存在系统的一个文件中，该文件中每一行的格式如下：

  • groupadd:创建新组
  • groupmod:修改组

sed 和 gawk

sed

sed 和 gawk是实现自动格式化、插入、修改或删除文本元素的简单命令行编辑器。

sed编辑器被称作流编辑器（stream editor），和普通的交互式文本编辑器恰好相反。在交互式 文本编辑器中（比如vim），你可以用键盘命令来交互式地插入、删除或替换数据中的文本。流编辑器则会在编辑器处理数据之前基于预先提供的一组规则来编辑数据流。

sed执行步骤：

  1. 一次从输入中读取一行数据。

  2. 根据所提供的编辑器命令匹配数据。

  3. 按照命令修改流中的数据。

  4. 将新的数据输出到STDOUT。

退出脚本

Linux提供了一个专门的变量$?来保存上个已执行命令的退出状态码。

常见状态码：

  • 0 ：命令正常执行并退出
  • 1 ：一般性未知错误
  • 2 ：不适合的shell命令
  • 126: 命令不可执行
  • 127: 没找到该命令
  • 128: 无效退出参数
  • 130: Ctrl+C终止的命令

结构化命令

if-then

if 后面的command被执行，如果退出状态码是0（成功运行），则会执行then后面的commands。fi表示if语句结束。

if-then-else

跟if-then基本一样，不同的是当command执行返回的状态码不是0（未能成功执行）的时候，会执行else后面的commands。

elif

执行command1，如果状态返回码为0，则执行commands；如果状态返回码不为0，则执行command2，如果command2返回0，则执行more commands，否则结束。

case

case命令会将指定的变量与不同模式进行比较。如果变量和模式是匹配的，那么shell会执行 为该模式指定的命令。

控制脚本

Linux利用信号与运行在系统中的进程进行通信
，包括利用信号来停止、启动、终止进程。

  • 中断进程：Ctrl+C
  • 暂停进程：Ctrl+Z
  • 杀死进程：kill -9 pid pid是进程号

方括号中的数字[1]是shell分配的作业号（job number）。shell中运行的每个进程称为作业， 并为每个作业分配唯一的作业号。它会给第一个作业分配作业号1，第二个作业号2，以此类推。

后台进程和终端会话联系在一起，如果终端会话退出，那么后台进程也会随之退出，如果希望运行在后台模式的脚本在登出终端后能够继续运行，则需要使用nohup命令。
